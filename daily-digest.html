Automation Project

Automated Daily Learning Digest

Julianne Toh · March 2026

Built with n8n, Claude API, Gmail

What it does

An automated system that delivers a personalized daily email every morning at 6am, combining curated news across scam/fraud policy, AI/ML product trends, and Singapore tech courses with structured learning tracks in PM concepts and probabilistic thinking. It also monitors specific sources like OGP's careers page and Dario Amodei's blog for changes.

Why I built it

During my job search, I needed to stay current across multiple domains (scam policy, AI product management, PM fundamentals) without spending hours each morning reading. I also wanted structured learning that would build knowledge cumulatively rather than randomly. Instead of subscribing to dozens of newsletters and hoping I'd read them, I built a system that did the filtering, synthesis, and scheduling for me.

How it works

An n8n workflow triggers daily and runs two parallel tracks. The first loads the day's PM concept and probabilistic thinking topic from JSON databases I designed (40 PM concepts across 8 weeks, 20 probabilistic thinking concepts across 10 weeks). The second sends a detailed prompt to Claude API via HTTP request, asking it to search the web for relevant news across my target categories and return structured JSON. A merge node combines both tracks, and a final code node assembles the HTML email and sends it via Gmail. The workflow includes day-of-week logic so different content categories appear on different days, preventing information overload.

What I learned

The hardest part wasn't the automation; it was the content design. Deciding what to include, what to exclude, and how to sequence learning concepts required more thought than the technical build. I also learned practical lessons about API timeout handling, timezone logic in scheduled workflows, and why string concatenation beats template literals in n8n's code nodes. Most importantly, the project reinforced the thesis of my writing piece: the value isn't in the tool, it's in understanding the workflow well enough to design something genuinely useful.
