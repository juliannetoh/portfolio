AI-Powered Resume Tailoring Pipeline

Julianne Toh · March 2026

Built with Python, Claude API, ReportLab, Cursor

What it does

A command-line tool that takes a job description and generates a tailored, ATS-friendly resume as a PDF. It maintains a single master data file containing all my career content (three versions of career summaries, role-specific bullets, technical skills, interview stories, portfolio items) and uses Claude API to select and reframe the most relevant content for each application.

Why I built it

Each job application I was targeting required a differently weighted resume. The scam policy role needed SPF front and centre. The AI PM role needed Amazon's ML work leading. Manually rewriting for each application was time-consuming and error-prone. I wanted a system where my career content lived in one place and the tailoring happened automatically, so I could focus on quality applications rather than formatting.

How it works

The master data file (resume_master_data.json) is the single source of truth, containing every bullet, summary variant, and skill I might need. The generator script supports preset versions (general, scamtech, pair) that select pre-configured content, plus a custom mode that accepts any job description. In custom mode, the script sends the job description and master data to Claude API with instructions to select the most relevant content, reorder sections, and adjust emphasis. The output is a formatted two-page PDF generated with ReportLab, with consistent typography, proper margins, and ATS-compatible formatting.

What I learned

Separating content from presentation was the key design decision. Once all my career data lived in structured JSON rather than in Word documents, tailoring became a selection problem rather than a rewriting problem. I also learned that AI-generated resumes need human review for tone; Claude's first drafts were sometimes too polished and lost my voice. The system works best as a first draft generator that I then edit, not as a fully autonomous pipeline. This mirrors the human-in-the-loop principle from my Amazon work: AI handles the routine selection and formatting, humans handle the judgment about voice and emphasis.
